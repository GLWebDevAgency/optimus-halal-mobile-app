## Task 1: SSR-Visible Animations — Fix First Paint

The #1 problem: all critical content renders `opacity: 0` at SSR. Fix `AnimateIn` and `SplitText` to support an `ssrVisible` mode where content is fully visible in HTML, and animations only enhance on scroll.

**Files:**
- Modify: `web/src/components/animations/animate-in.tsx`
- Modify: `web/src/components/animations/split-text.tsx`

- [ ] **Step 1: Update AnimateIn with ssrVisible prop**

Add a prop `ssrVisible?: boolean` that when true, sets `initial="visible"` instead of `initial="hidden"`. The animation still triggers on scroll for non-critical elements. For critical elements (hero H1, subtitle, CTA, stats), we use `ssrVisible` so the HTML renders visible.

```tsx
// In AnimateIn component:
// Add ssrVisible prop to interface
// Change initial: ssrVisible ? "visible" : "hidden"
// When ssrVisible, animate is always "visible" — animation is purely decorative
```

- [ ] **Step 2: Update SplitText with ssrVisible prop**

Add `ssrVisible?: boolean` that renders all characters with `initial={{ opacity: 1, rotateX: 0, y: 0 }}` — content visible immediately, animation decorative only.

- [ ] **Step 3: Verify SSR output**

Run: `cd web && pnpm build`
Expected: Build passes. The hero H1, subtitle, CTA, stats, and trust bar render visible in the static HTML.
